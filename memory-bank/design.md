# 네이버 타임 클릭 매크로 설계 문서

---

## 1. 프로젝트 개요

네이버 서버 시간을 기준으로 사용자가 지정한 특정 시각에 미리 등록한 3개의 화면 좌표를 자동으로 클릭하는 매크로 프로그램이다. 오픈런, 선착순 이벤트, 티켓팅 등 정확한 시각에 클릭이 필요한 상황에서 사용한다.

---

## 2. 동작 방식

```
프로그램 실행
    └─> 네이버 서버 시간 동기화 (HTTP HEAD 요청)
        └─> GUI 화면 표시
            └─> 사용자: 실행 시각 설정 (시/분/초/밀리초)
                └─> 사용자: 클릭 좌표 3개 등록 (Z 키 입력 방식)
                    └─> 매크로 시작 버튼 클릭
                        └─> 설정 시각 도달 시 좌표 순서대로 자동 클릭
                            └─> 완료 알림
```

1. **실행** - 프로그램 시작 시 네이버 서버와 로컬 시계의 오차를 계산하여 내부적으로 보정값(offset)을 저장한다.
2. **시간 설정** - 사용자가 GUI에서 매크로를 실행할 목표 시각을 입력한다.
3. **좌표 등록** - 마우스를 원하는 위치에 올려놓고 Z 키를 누르면 해당 순서의 좌표가 등록된다.
4. **매크로 시작** - 시작 버튼을 누르면 프로그램이 보정된 서버 시간을 기준으로 대기하다가 목표 시각에 등록된 좌표 3곳을 순서대로 클릭한다.

---

## 3. 기능 요구사항

### 3-1. 네이버 서버 시간 확인 기능

- 프로그램 실행 시 `https://www.naver.com` 에 HTTP HEAD 요청을 보내고 응답 헤더의 `Date` 필드를 파싱하여 서버 시각을 획득한다.
- 획득한 서버 시각과 로컬 시스템 시각의 차이(ms 단위)를 오프셋으로 저장한다.
- 이후 모든 시간 판단은 `로컬시각 + 오프셋 = 보정 서버시각` 공식으로 계산한다.
- 사용자가 수동으로 ms 단위 오프셋을 조정할 수 있다.
- 오프셋 조정 중 실시간으로 "네이버 서버 시간과의 일치율(%)"을 화면에 표시한다.

#### 일치율 계산 방식

```
허용 오차 범위 = ±500ms (기준값)
현재 오프셋 오차 = |자동측정 오프셋 - 사용자 조정 오프셋|
일치율(%) = max(0, (1 - 현재 오프셋 오차 / 허용 오차 범위) × 100)
```

예시: 자동측정 오프셋이 120ms이고 사용자가 170ms로 조정한 경우
- 현재 오프셋 오차 = |120 - 170| = 50ms
- 일치율 = (1 - 50 / 500) × 100 = **90%**

### 3-2. 시간 설정 기능

- 목표 실행 시각을 시(hour), 분(minute), 초(second), 밀리초(millisecond) 단위까지 설정한다.
- 설정한 시각까지 남은 시간을 실시간 카운트다운으로 표시한다.
- 목표 시각이 현재보다 과거인 경우 경고 메시지를 표시하고 매크로 시작을 차단한다.

### 3-3. 좌표 지정 기능

- 등록 버튼을 클릭하면 좌표 등록 모드로 진입한다.
- 등록 모드에서 사용자가 마우스를 원하는 위치에 올려놓고 **Z 키**를 누르는 순간의 마우스 좌표(x, y)를 해당 순번의 좌표로 저장한다.
- 3개의 좌표를 순서대로 등록한다 (좌표1 → 좌표2 → 좌표3).
- 등록된 좌표는 화면에 `좌표N: (x, y)` 형식으로 표시한다.
- 이미 등록된 좌표는 개별적으로 재등록(수정)할 수 있다.

## 4. 기술 스택

| 항목 | 선택 기술 | 사용 이유 |
|------|-----------|-----------|
| 언어 | Python 3.10+ | 빠른 개발, 풍부한 라이브러리 |
| GUI | `tkinter` | 표준 라이브러리, 별도 설치 불필요 |
| 마우스 자동 클릭 | `pyautogui` | 크로스 플랫폼 마우스/키보드 제어 |
| 키보드 감지 (Z 키) | `pynput` | 전역 키보드 리스너, 포커스 무관 |
| HTTP 요청 | `requests` | 네이버 서버 시간 획득 |
| 시간 처리 | `datetime`, `time` | 정밀 시각 계산 |

## 5. 구현 세부사항

### 5-1. 네이버 서버 시간 동기화

```python
# 핵심 로직 흐름 (의사 코드)
before = time.time()
response = requests.head("https://www.naver.com")
after = time.time()

server_time = parse_http_date(response.headers["Date"])  # RFC 1123 파싱
request_latency = (after - before) / 2                   # 왕복 지연의 절반
offset_ms = (server_time - after + request_latency) * 1000
```

- 네트워크 왕복 지연을 절반으로 나눠 편도 지연을 보정한다.
- 동기화는 프로그램 시작 시 1회 자동 수행하며, 사용자가 수동 재동기화 버튼으로 재실행할 수 있다.

### 5-2. 정밀 시각 대기 및 클릭

```python
# 핵심 로직 흐름 (의사 코드)
while True:
    current_corrected = time.time() * 1000 + offset_ms
    if current_corrected >= target_time_ms:
        break
    remaining = target_time_ms - current_corrected
    if remaining > 50:
        time.sleep(0.01)   # 10ms 슬립으로 CPU 점유 최소화
    # 50ms 이내 구간은 busy-wait으로 정밀도 확보

pyautogui.click(coord1[0], coord1[1])
pyautogui.click(coord2[0], coord2[1])
pyautogui.click(coord3[0], coord3[1])
```

- 목표 시각 50ms 전까지는 슬립으로 CPU를 절약하고, 50ms 이내 구간은 busy-wait으로 클릭 타이밍 정밀도를 높인다.
- `pyautogui.PAUSE = 0`으로 설정하여 클릭 사이 기본 지연을 제거한다.

### 5-3. Z 키 전역 감지 (pynput)

```python
# 핵심 로직 흐름 (의사 코드)
from pynput import keyboard, mouse

def on_press(key):
    if key == keyboard.KeyCode.from_char('z') and registering:
        pos = mouse.Controller().position
        save_coordinate(current_slot, pos)

listener = keyboard.Listener(on_press=on_press)
listener.start()
```

- `pynput`의 전역 리스너를 사용하므로 다른 창이 포커스를 가지고 있어도 Z 키 입력을 감지한다.
- 좌표 등록 중임을 나타내는 상태 플래그(`registering`)로 의도치 않은 등록을 방지한다.

## 6. UI 레이아웃

```
┌─────────────────────────────────────────────────┐
│         네이버 타임 클릭 매크로                  │
├─────────────────────────────────────────────────┤
│ [서버 시간 동기화]  서버시간: 14:59:58.320       │
│ 오프셋 조정: [──●────────] +120 ms              │
│ 네이버 시간 일치율: ████████░░ 90%              │
│                          [재동기화]              │
├─────────────────────────────────────────────────┤
│ 목표 실행 시각                                   │
│  시: [15] 분: [00] 초: [00] ms: [000]           │
│ 남은 시간: 00:00:01.680                         │
├─────────────────────────────────────────────────┤
│ 클릭 좌표 등록                                   │
│  좌표1: (1024, 512)   [재등록]                  │
│  좌표2: ( 980, 400)   [재등록]                  │
│  좌표3: (미등록)      [등록]                    │
│                                                 │
│  * 등록 버튼 클릭 후 마우스를 원하는 위치에      │
│    올려놓고 Z 키를 누르세요.                     │
├─────────────────────────────────────────────────┤
│              [매크로 시작]                       │
│         상태: 대기 중 / 실행 중 / 완료           │
└─────────────────────────────────────────────────┘
```

### UI 구성 요소 설명

- **서버 시간 동기화 영역** - 현재 보정된 네이버 서버 시각을 실시간으로 표시하고, 슬라이더로 ms 단위 오프셋을 수동 조정할 수 있다. 조정값에 따라 일치율 프로그레스 바가 실시간 업데이트된다.
- **목표 실행 시각 영역** - 시/분/초/밀리초를 개별 입력 필드로 구성하며, 남은 시간을 `HH:MM:SS.mmm` 형식으로 카운트다운한다.
- **클릭 좌표 등록 영역** - 3개의 좌표 슬롯을 표시하고 각각 개별 등록 및 재등록이 가능하다. 등록 모드 진입 시 해당 행이 강조 표시된다.
- **매크로 시작 버튼** - 3개 좌표가 모두 등록되고 목표 시각이 유효할 때만 활성화된다. 실행 중에는 취소 버튼으로 전환된다.

## 8. 예외 처리 및 주의사항

- 네이버 서버 요청 실패 시 로컬 시간으로 폴백하고 사용자에게 경고를 표시한다.
- `pyautogui`의 페일세이프(마우스를 화면 모서리로 이동 시 중단) 기능은 기본 활성화 상태를 유지한다.
- 목표 시각이 이미 지난 경우 매크로 시작을 차단하고 안내 메시지를 표시한다.
- Windows 환경에서 `pynput` 전역 리스너는 관리자 권한 없이도 동작하나, 일부 보안 프로그램이 차단할 수 있음을 안내한다.
